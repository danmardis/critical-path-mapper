# Domain Strategy

## Recommendation

Host Critical Path Mapper under a professional personal domain, not Hard To Finder.

Recommended initial URL:

```text
danielmardis.com/critical-path-mapper
```

Alternative:

```text
projects.danielmardis.com/critical-path-mapper
```

## Why not Hard To Finder?

`hardtofinder.com` is better suited for vintage research, discovery, collecting, resale, internal tools, and niche artifact projects.

Critical Path Mapper is a public career/portfolio project. It should reinforce a professional identity around systems thinking, business systems, operations, software, data, and workflow design.

Using Hard To Finder for everything would dilute both brands:

- Hard To Finder becomes less focused.
- Critical Path Mapper looks less professional and less directly tied to the portfolio story.

## Why use a personal domain first?

A personal-domain subpath is the best starting point because it:

- connects the project to the public portfolio,
- avoids buying domains too early,
- keeps maintenance simple,
- reduces brand fragmentation,
- works well for a portfolio project,
- and keeps the app flexible if it later becomes a product.

## When to buy a standalone domain

Consider a separate domain only if the app becomes a real product with:

- public users,
- signups,
- newsletter traffic,
- paid features,
- external contributors,
- or a distinct brand identity.

Possible standalone domains to evaluate later:

- `criticalpathmapper.com`
- `criticalpath.app`
- `pathmapper.app`
- `fasttrackplanner.com`
- `triagepath.com`

Do not buy domains just because the project feels exciting on day one.

## Suggested hosting path

### Phase 1: GitHub Pages or Vercel preview

Use the simplest static deployment that gets the app online quickly.

### Phase 2: Personal portfolio route

Add the project at:

```text
danielmardis.com/critical-path-mapper
```

or:

```text
projects.danielmardis.com/critical-path-mapper
```

### Phase 3: Standalone domain only if validated

Move to a standalone project domain only after the app has evidence of public value.

## Portfolio positioning

The domain should support this message:

> I build practical systems that turn ambiguous work into clear action.

Critical Path Mapper should appear as part of a broader professional portfolio, not as a random side quest floating in domain space.

## Current decision

Default to:

```text
danielmardis.com/critical-path-mapper
```

Use a standalone domain later only if the app grows beyond portfolio-project status.
